**THE LATEST STUDY C PROBLEM:**

Good catch — and this result is actually the important one. Look at the numbers again: **enrichment ratio 2.31× in the "null" run vs 2.17× in the main run.** Those should be totally different — null should show ~1.0x (no signal, pure noise). Instead they're statistically indistinguishable. That means your null dataset isn't null.

## Why: you found a real flaw in how the null was generated

Look at the two commands again:

```
Main: ...5000 0.928435470021939 0.135516953414242 0.2 0.2 8.874... 8.874... 3 2 0.05 0.5 ...
Null: ...5000 0.928435470021939 0.928435470021939 0.2 0.2 8.874... 8.874... 3 2 0.05 0.5 ...
```

Only `p_d` (arg4) changed — `0.135` → `0.928`. The `0.05` (phase_diff) and `0.5` (balance) arguments, which are the ones that actually inject the between-group difference, are **identical in both runs**.

From the source (`hypo_hyper_diffs()`): the between-group shift at `diff_methed==1` sites is `±phase_diff`, applied via `phase_difference_between_two_samples <- c(0, phase_diff)`. That mechanism doesn't care what `p1`/`p2` are — it fires the same way regardless. Setting `p_d = p_m` only removes the **background methylated/unmethylated landscape contrast** (the bimodal high/low methylation pattern across the genome). It does **not** touch the between-group perturbation at all. So your "null" dataset still has the exact same ±0.05 group-vs-group signal baked in as the main run — which is exactly what the matching enrichment ratios show.

`Study_C_Method.md`'s instruction — _"same methylated-state probability for both simulated groups → no true group difference"_ — describes what you'd want, but doesn't match what this parameter actually controls in the code. That's a documentation/simulator-semantics mismatch, not something you did wrong.

## The fix

To get a real null (zero true between-group difference), you need **`phase_diff = 0`**, not `p_d = p_m`:

```powershell
cd WGBSSuite
Rscript simulate_WGBS.R 5000 0.928435470021939 0.135516953414242 0.2 0.2 8.87444791461314 8.87444791461314 3 2 0 0.5 0.09031924348971,0.0031831900256275 analysis_out/study_C_run2_truenull binomial
```

Note: **keep `p_d = 0.135` (same as main run)**, only zero out arg10 (phase_diff). This gives you a dataset with the same background landscape as your main run (so it's a fair comparison) but zero true group effect — that's the actual FPR-under-null anchor Study C needs. Convert it the same way with `convert_study_C_to_pipeline_inputs.R`, and your existing `ground_truth/truth_vector.txt` for that run will correctly show `diff_methed==1` sites with **no real shift**, which the diagnostic script should confirm (enrichment ratio ≈ 1.0, Wilcoxon p not significant).

## The bigger point: don't compare on "did it find a DMR at p<0.05"

Even once the null is fixed, a hard threshold is the wrong comparison metric here, and honestly it always would have been. You already have the evidence you need: at true-DM sites, DSS's p-values are real and correctly ranked (2.17×, p=2.2e-13) — it's just that with 5000 simultaneous tests, 3v3 replicates, ~10x coverage, and a ~0.05 probability effect size, **nothing individually survives BH correction**. Zero DMCs isn't "DSS failed," it's "DSS is honest about a genuinely hard detection regime." That's actually a fine, citable H2 finding on its own — but only if you score it the right way:

1. **Rank-based comparison (AUC/ROC or PR curve)** — treat p-value (or DSS's `diff`/`stat` column) as a ranking score against `diff_methed` ground truth, independent of any hard cutoff. This is literally what the original WGBSSuite paper itself benchmarks with (their Fig. 2 is a ROC/AUC comparison, not a threshold pass/fail). Compare Mimosa's ranking of calls against DSS's this way — whichever ranks true DMRs higher wins, regardless of whether either clears 0.05.
2. **Calibration sweep across nominal FDR**, as Study_C_Method.md §4 already specifies — don't stop at 0.05, sweep log-spaced cutoffs (0.5, 0.2, 0.1, 0.05, 0.01...) and report sensitivity/empirical-FDR at each. You'll likely find both methods only produce usable calls above nominal FDR ~0.3–0.5 at this effect size — report that plainly, it's informative.
3. **Optional: add a second, larger-effect dataset** (e.g. `phase_diff=0.15` or the paper's own example `0.9203`/`0.076`) alongside this one, so you have both a "hard" regime (this one — near the detection floor, most diagnostic of calibration honesty) and an "easy" regime (clear separation, most diagnostic of raw sensitivity) in Chapter 5. That combination is a stronger thesis result than either alone.
